# Task 02 — Job Data Model & CRUD

## 목표
더미 데이터를 실제 저장 구조로 바꾸고, 작업 생성 / 조회 흐름을 만든다.

## 이번 단계에서 구현할 것
- SQLite 또는 간단한 로컬 DB 연결
- Job 데이터 모델 정의
- 작업 생성 기능
- 작업 목록 조회 기능
- 작업 상세 조회 기능
- 상태 필드 및 타임스탬프 저장
- 기본 입력 검증

## Job 필드
- id
- title
- engine
- prompt
- status
- createdAt
- updatedAt
- startedAt
- finishedAt
- resultSummary
- previewImagePath
- logPath
- errorMessage

## 구현 범위
### 백엔드
- DB 초기화 코드
- Job 테이블 생성
- 작업 생성 함수
- 작업 목록 조회 함수
- 작업 상세 조회 함수

### 프론트엔드
- 새 작업 폼 제출
- 작업 생성 후 상세 또는 목록으로 이동
- 작업 목록을 DB 기준으로 렌더링
- 작업 상세 페이지에서 실제 DB 데이터 표시

## 입력 검증
최소 검증:
- title 필수
- engine 필수
- prompt 필수
- 너무 긴 빈 문자열/공백 입력 방지

## 산출물
- 실제 저장 가능한 작업 CRUD 기초
- 작업 생성 후 목록/상세에서 확인 가능
- 더미 데이터 제거 또는 최소화

## 하지 않을 것
- CLI 실행
- 로그 스트리밍
- 텔레그램 전송

## 완료 기준
- 새 작업을 만들 수 있어야 한다.
- 만든 작업이 목록에 보여야 한다.
- 작업 상세에서 prompt와 상태를 볼 수 있어야 한다.
- 새로고침해도 데이터가 유지되어야 한다.

## 코덱스 실행 프롬프트
```text
project_plan.md와 tasks/02_jobs.md를 읽고 작업 데이터 저장 구조를 구현해줘.
SQLite를 연결해서 Job 모델을 만들고, 작업 생성 / 목록 / 상세 조회가 가능하게 해줘.
기본 입력 검증도 넣고, UI는 기존 페이지를 유지하면서 실제 DB 데이터를 사용하도록 바꿔줘.
변경 파일 목록과 다음 단계 제안도 함께 정리해줘.
```
