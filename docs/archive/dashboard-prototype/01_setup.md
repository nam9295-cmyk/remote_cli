# Task 01 — Setup & App Skeleton

## 목표
프로젝트의 기본 뼈대를 만든다.
이 단계에서는 실제 CLI 실행이나 텔레그램 연동은 하지 않고, 화면 구조와 개발 기반만 만든다.

## 이번 단계에서 구현할 것
- Next.js 앱 초기 세팅
- Tailwind CSS 적용
- 기본 라우트 생성
- 공통 레이아웃 생성
- 더미 데이터 기반 작업 리스트 표시
- 기본 컴포넌트 생성
- 모바일에서도 보기 좋은 반응형 UI 뼈대

## 생성해야 할 페이지
- `/` : 홈 또는 대시보드
- `/jobs` : 작업 목록
- `/jobs/[id]` : 작업 상세
- `/new` : 새 작업 생성
- `/settings` : 설정 페이지

## 생성해야 할 공통 컴포넌트
- `JobCard`
- `StatusBadge`
- `SectionHeader`
- `EmptyState`

## 더미 데이터 예시
작업 데이터는 임시 배열로 구성한다.
필드:
- id
- title
- engine
- status
- createdAt
- updatedAt
- resultSummary

상태 예시:
- queued
- running
- success
- failed

## 디자인 방향
- 심플하고 모던한 대시보드
- 카드형 리스트
- 상태 배지 강조
- 모바일 우선
- 너무 화려한 효과보다 안정적인 정보 구조 우선

## 산출물
- 실행 가능한 Next.js 앱
- 기본 페이지 라우트
- 더미 데이터 기반 UI
- 공통 컴포넌트 분리

## 하지 않을 것
- SQLite 연결
- 실제 CLI 실행
- 텔레그램 연동
- 로그인/권한 관리

## 완료 기준
- 페이지 이동이 가능해야 한다.
- 홈과 작업 목록에서 더미 작업 리스트가 보여야 한다.
- 작업 상세 페이지에서 특정 작업 정보를 볼 수 있어야 한다.
- 새 작업 페이지와 설정 페이지는 폼 UI 뼈대가 있어야 한다.

## 코덱스 실행 프롬프트
```text
project_plan.md와 tasks/01_setup.md를 읽고 setup 단계만 구현해줘.
Next.js + Tailwind 기반으로 홈, 작업목록, 작업상세, 새 작업, 설정 페이지를 만들어줘.
더미 데이터로 먼저 동작하게 하고, 공통 컴포넌트(JobCard, StatusBadge 등)를 분리해줘.
모바일에서도 보기 좋게 구성하고, 변경 파일 목록과 다음 단계 제안도 함께 정리해줘.
```
