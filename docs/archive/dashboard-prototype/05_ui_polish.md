# Task 05 — UI Polish & Mobile Optimization

## 목표
기능이 붙은 상태에서 화면 완성도와 사용성을 높인다.

## 이번 단계에서 구현할 것
- 모바일 레이아웃 개선
- 카드 간격 / 타이포 / 계층 구조 정리
- 상태 배지 시각성 강화
- 로그 뷰어 가독성 개선
- 빈 상태 / 로딩 상태 / 에러 상태 UI 추가
- 설정 페이지 정보 구조 정리
- 작업 상세 결과 영역 개선

## 개선 포인트
### 홈 / 목록
- 카드 크기와 정보 우선순위 정리
- 최신 작업이 잘 보이게 구성
- 상태별 시각 구분 강화

### 새 작업
- 폼 입력 구조 정리
- 엔진 선택 UI 단순화
- 잘못된 입력 시 에러 메시지 명확화

### 작업 상세
- 로그 / 결과 / 에러 구역 구분
- 긴 로그 접기/펼치기
- 결과 요약을 상단에 우선 배치
- 모바일에서 스크롤 피로 줄이기

### 설정
- 환경변수 안내
- 저장 경로 안내
- 엔진 설정 설명

## UX 상태 추가
- Empty state
- Loading skeleton 또는 loading state
- Error state
- Success feedback

## 산출물
- 더 정돈된 대시보드 UI
- 모바일 사용성 개선
- 상태별 피드백 강화

## 하지 않을 것
- 대규모 리브랜딩
- 애니메이션 과다 사용
- 기능 구조 대폭 변경

## 완료 기준
- 모바일에서도 작업 생성/조회/상세 확인이 편해야 한다.
- 빈 화면 / 실패 / 로딩 상태가 모두 자연스럽게 보여야 한다.
- 정보 계층이 분명해서 대표가 봐도 빠르게 이해 가능해야 한다.

## 코덱스 실행 프롬프트
```text
project_plan.md와 tasks/05_ui_polish.md를 읽고 UI 완성도를 높여줘.
모바일 최적화, 상태 배지, 로그 뷰어, 빈 상태/로딩/에러 상태를 정리하고,
작업 목록/상세/설정 화면의 정보 계층을 더 명확하게 다듬어줘.
기능 구조는 크게 바꾸지 말고 UI/UX 중심으로 정리해줘.
변경 파일 목록과 다음 단계 제안도 함께 정리해줘.
```
