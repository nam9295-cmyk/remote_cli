# Task 04 — Telegram Notification Integration

## 목표
작업 완료/실패 결과를 텔레그램 채팅방으로 자동 전송한다.

## 이번 단계에서 구현할 것
- Telegram Bot API 연동
- 환경변수 기반 설정
- 작업 성공 알림 전송
- 작업 실패 알림 전송
- 메시지 포맷 정리
- 알림 전송 이력 저장

## 필요한 환경변수
- TELEGRAM_BOT_TOKEN
- TELEGRAM_CHAT_ID
- PUBLIC_BASE_URL

## 전송 메시지 구성
### 성공 메시지 예시
- 작업 제목
- 엔진
- 상태
- 결과 요약
- 결과 링크

### 실패 메시지 예시
- 작업 제목
- 엔진
- 상태
- 에러 요약
- 상세 링크

## 추가 가능 항목
- preview 이미지 첨부
- 실행 시간
- 생성 시각

## NotificationLog 필드
- id
- jobId
- channelType
- sentAt
- status
- messageId

## UI 반영
- 작업 상세 페이지에 텔레그램 전송 상태 표시
- 최근 전송 이력 표시 가능하면 추가

## 산출물
- 작업 완료/실패 시 텔레그램 메시지 자동 전송
- 실패 시에도 기록 남김
- 메시지 포맷 정리

## 하지 않을 것
- 텔레그램에서 직접 명령 수신
- 다중 채팅방 라우팅
- 사용자별 권한 분리

## 완료 기준
- 성공한 작업이 텔레그램으로 전송되어야 한다.
- 실패한 작업도 실패 메시지가 가야 한다.
- 전송 성공/실패 여부가 기록되어야 한다.
- 상세 페이지에서 전송 여부를 확인할 수 있어야 한다.

## 코덱스 실행 프롬프트
```text
project_plan.md와 tasks/04_telegram.md를 읽고 텔레그램 알림 기능을 구현해줘.
작업 성공/실패 시 TELEGRAM_BOT_TOKEN, TELEGRAM_CHAT_ID를 사용해 메시지를 자동 전송하고,
NotificationLog도 저장해줘.
작업 상세 페이지에서 알림 전송 여부를 확인할 수 있게 해줘.
변경 파일 목록과 다음 단계 제안도 함께 정리해줘.
```
